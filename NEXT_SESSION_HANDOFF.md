# Next Session Handoff

다음 작업 때 이 문서만 보고 바로 이어서 개발할 수 있도록 정리한 체크리스트입니다.

## 0) 시작 전 3분 체크

1. `.env.local` 확인
   - `MONGODB_URI=mongodb://localhost:27017/mentorlink`
2. 개발 서버 실행
   - `npm run dev`
3. 로그인 테스트 계정 확인
   - 필요 시 `npm run seed`로 샘플 데이터 재주입

---

## 1) 현재 완료 상태 (요약)

- 인증: 회원가입/로그인/내정보/역할전환 완료
- 강의: 목록/상세/수강신청/내 수강 내역/상태변경(완료/취소) 완료
- 멘토: 목록/상세/세션 예약/내 세션/상태변경(완료/취소) 완료
- 커뮤니티: 가입 신청/내 신청 내역 완료
- 프리랜서: 지원 신청/내 신청 내역 완료
- 관리자: 검수 큐 조회 + 승인/거절 + 사용자 페이지 요약 위젯 완료
- 대시보드: 실데이터 연동 + 내 활동 바로가기 + 최근 목록 + 상태 액션 완료
- 디자인: `StatusBadge`, `LoadingState`, `EmptyState` 기반 통일 1차 완료
- i18n: 헤더 + 대시보드 주요 텍스트 2차 반영 완료

완료 기준:

## C. 결제 전 단계 정책 고정 (3순위)

- 항목:
  - enrollment 상태 전이표
  - session 상태 전이표
  - admin 승인 정책

완료 기준:
- 팀원이 봐도 상태 정책이 모호하지 않음

## 3) 다음 구현 후보 (결제 직전 단계)

1. `my/enrollments`에 결제상태 표시 개선 (paid/pending/refunded 라벨 UX)

## 4) 결제 연동 진입 전 준비물

결제 붙이기 전에 아래 4개를 먼저 확정:

1. PG 선택 (국내/해외)
2. 환불 정책 (전액/부분/시간 기준)
3. 영수증 포맷 필드 (주문번호, 공급가, 부가세 등)
4. 정산 주기 (주간/월간)
